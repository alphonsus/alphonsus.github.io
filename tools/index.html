Tools

Here is a list of tools and resources I use for making games.

Game Development

  • HaxeFlixel – Main game development toolkit. Basically, it is Haxe programming language + OpenFL + Flixel game framework.
  • Unity – Used to be my main game engine, but switched to HaxeFlixel very recently.
  • Sublime Text 3 – IDE. I prefer a lightweight lightning fast text editor over feature-packed but slow IDE’s any day. You just need to get used to keyboard shortcuts and install the necessary packages.
  • Tiled – Tiled map level editor. Usually export to CSV files which are then fed into HaxeFlixel as tilemap objects.

 

Art

  • Pyxel Edit – In-game pixel art sprites.
  • Photoshop – Non in-game stuff, like making game title banners for marketing.

 

Project Management

  • Git – Version control system.
  • GitHub – Git repository host.
  • iTerm 2 – My terminal where I do version control with Git or compile Haxe projects among many other UNIX-related tasks.
  • Trello – Used as a “kanban” board typically with three columns: “Todo”, “In Progress”, and “Done.” We also label the tasks using color labels, i.e. green for feature, orange for art-related stuff, red for bugs, etc.
  • Slack – The best app for communication. It’s more than just an IM app – we integrated it with GitHub so pushing to remote repositories will ping a notification to everyone in the chat room. We also use this to share images and screenshots which you can “pin” for quick references later.

 

Audio

  • BFXR – Chiptune sound effects.
  • FreeMusicArchive - Where I look for free background music for our games.
  • LMMS – Open source digital audio workstation.
